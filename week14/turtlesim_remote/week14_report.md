## 1. 项目概述

本项目实现了一个手机遥控 turtlesim 迷宫探索系统。用户通过手机浏览器连接 Tailscale 虚拟网络，向 WSL 中的 ROS2 turtlesim 发送控制命令，驱动小乌龟在迷宫中移动并到达终点。

核心工程原则：**网络接收与机器人控制写在同一个常驻程序中**。手机网页只负责发送命令，不直接操控机器人。

## 2. 系统架构

```
┌──────────────────────┐
│   手机浏览器           │
│   index.html         │
│   长按方向键发送指令    │
└──────────┬───────────┘
           │ WebSocket (Tailscale 组网)
┌──────────▼───────────┐
│ turtlesim_web_bridge  │  ← 单一常驻程序
│ ├── aiohttp WebSocket │
│ ├── ROS2 Node         │
│ ├── 碰撞检测           │
│ └── /turtle1/cmd_vel  │
└──────────┬───────────┘
           │ ROS2 Topic
┌──────────▼───────────┐
│   turtlesim_node      │
│   (ROS2 仿真器)       │
└──────────────────────┘
```

## 3. 迷宫设计

迷宫由 `maze.py` 生成，使用递归回溯算法生成 5×5 网格，并通过编织 (braid) 技术随机打通 28% 的额外墙壁，制造环路使迷宫有多条通路。

- 网格尺寸: 5×5，每格 1.7m
- 起点: 左下角 (0,0)
- 终点: 右上角 (4,4)
- 障碍物: 自动生成的墙体矩形
- 碰撞检测: 考虑小乌龟半径 (0.45m) 的膨胀边界

## 4. 核心代码改动

### turtlesim_web_bridge.py

在课程 starter 代码基础上：
- 实现了完整的 ROS2 节点 (`TurtleWebBridge`)
- 订阅 `/turtle1/pose` 获取实时位置
- 发布 `/turtle1/cmd_vel` 驱动小乌龟
- `compute_safe_motion()` 方法实现碰撞预判：根据当前速度和方向预测下一步位置，若会碰到边界或障碍物则拦截

### index.html

- 方向键使用 `touchstart`/`touchend` 事件实现长按控制、松开停止
- Canvas 实时绘制迷宫边界、障碍物、起点(绿)、终点(黄)、小乌龟位置和朝向
- 显示实时遥测数据：坐标、速度、阻挡状态

## 5. 遇到的问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 手机无法访问页面 | 未在同一 Tailscale 网络 | `tailscale ip -4` 确认 WSL 地址 |
| 小乌龟不动 | 未 source ROS2 环境 | 每个终端都要 `source /opt/ros/humble/setup.bash` |
| 连接频繁断开 | 手机锁屏导致 WebSocket 断开 | 保持屏幕常亮 |
